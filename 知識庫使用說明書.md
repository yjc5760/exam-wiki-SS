## 1. 知識庫架構一覽

知識庫分為**七個知識層**，前三層自動生成，後四層由 Cowork 直接維護：

| 層 | 目錄 | 內容 | 生成方式 |
|----|------|------|---------|
| Layer 1 | `wiki/concepts/` | 核心概念（60 頁） | Cowork |
| Layer 1 | `wiki/problems/` | 98 道題解析頁 | Cowork |
| Layer 2 | `wiki/philosophy/` | 設計哲學（11 頁）：LRFD/ASD 各題型解題邏輯 | Cowork |
| Layer 3 | `wiki/methods/` | 解題方法論（18 頁）：步驟化計算流程 | Cowork |
| Layer 4 | `wiki/diagnosis/` | 題型診斷決策樹（8 頁） | Cowork |
| Layer 5 | `wiki/failure-modes/` | 失敗模式（4 頁）：強度/穩定/使用性/接合 | Cowork |
| Layer 6 | `wiki/materials/` | 材料行為（4 頁）：殘留應力/斷裂韌性/銲接性/應力應變 | Cowork |
| Layer 7 | `wiki/code-ref/` | 規範推導（16 頁）：係數/邊界值的來源推導 | Cowork |

**主導航入口：** `wiki/index.md`（七層全覽）、`wiki/by-year.md`（依考年）

## 2. 核心工作流程

```
【解析新考題】
  Cowork：「解析 XXXX 年考卷」
    ↓ Cowork 掃描 question_index.json，建立題目資料夾
  你把題目附圖截圖（fig-N.png）存入對應資料夾
    ↓
  告知 Cowork：「截圖完成，請開始解題」
    ↓ Cowork 一次解一題，存 .md 檔 + viz.html
  你驗算後告知 Cowork 更新 verificationStatus = "verified"
    ↓
  Cowork：ingest SS-XXXX-N  →  wiki 自動更新

【新增知識點】
  Cowork：「把這個知識點加入知識庫：[說明內容]」
    ↓ 自動判斷層別（methods / concepts / code-ref）
  建立 .md 檔 + 更新對應 index.md

【複習】
  直接問 Cowork 問題，或開啟 wiki/ 目錄瀏覽
```

## 3. Cowork 指令速查：解析與備考

### 3.1 解析考題（SOLVE）

```
解析 XXXX 年考卷          ← 掃描全年，建立題目資料夾
解析 SS-XXXX-N            ← 直接解單題（需已有附圖）
這題做完了，繼續下一題      ← 觸發解析下一題
```

注意：Cowork 一次只解一題；`raw/` 目錄所有原始檔案唯讀（`question_index.json` 除外）。

---

### 3.2 新增知識點（add-to-wiki）

```
把這個知識點加入知識庫：[說明]
新增知識點：[說明]
把 X 加進 wiki
新增方法頁 / 新增概念頁 / 新增推導頁
```

Cowork 自動判斷放哪層：

| 知識點類型 | 目錄 | 命名格式 |
|-----------|------|---------|
| 「**怎麼算**」的步驟流程 | `wiki/methods/` | `全小寫-連字號.md` |
| 「**是什麼/為什麼**」的概念定義 | `wiki/concepts/` | `全大寫-連字號.md` |
| 「**規範數字/邊界值**」的推導來源 | `wiki/code-ref/` | `全小寫-連字號.md` |

---

### 3.3 變數層次分析（VHA）

```
幫 SS-XXXX-N 做 VHA
SS-XXXX-N 套用變數層次分析
```

在題解 `§3` 後插入 `§3.5 變數層次分析`，將解題變數分三層：

| 層次 | 內容 | 複習策略 |
|------|------|---------|
| **L1** | 題目直接給定的數值 | 跳過 |
| **L2** | 需套公式推導的中間變數 | 標記 `⚠` 不熟的 |
| **L3** | 控制 L2 應用的深層概念 | 優先補強 `⚠⚠` |

---

### 3.4 VHA 藏寶圖儀表板（vha-treasure-map）

```
幫 SS-XXXX-N 做藏寶圖儀表板
VHA 轉化為儀表板
VHA dashboard SS-XXXX-N
```

將 `§3.5 VHA` 轉化為互動式 HTML 儀表板，儲存至 `wiki/queries/SS-YYYY-N-treasure-map.html`：

| 功能 | 說明 |
|------|------|
| KaTeX 數學式渲染 | 公式正確顯示，不出現 raw LaTeX |
| 變數顏色區分 | L1 綠 / L2 藍 / 本步求解琥珀框 / 最終未知紅框 |
| 知識線索卡片 | L3 知識點可互連，支援自評（掌握／卡關／待複習） |
| 陷阱警示 | 每步驟的 ⚠️ 陷阱以紅色框標示 |

> **前置條件：** 目標題目需已有完整 §3.5 VHA 區塊

---

### 3.5 備考分析指令

```
review [標籤]               ← 針對主題生成複習題組（如 review LTB側扭挫屈）
predict                     ← 根據歷年頻率預測今年可能出的考點
exam-sim YYYY               ← 模擬某年考試（只出題，不給答案）
compare SS-XXXX-N SS-YYYY-M ← 並排比較兩題的考點與陷阱差異
analyze YYYY                ← 分析單年考卷四題的考點分布與特色
study [主題/topicId]         ← 系統性單元專題複習（核心公式 + 歷年題 + 陷阱）
find [關鍵字]               ← 全庫關鍵字搜尋（如 find 塊狀剪力）
related SS-XXXX-N           ← 找出與某題考點相似的其他題目
```

## 4. Cowork 指令速查：維護與查詢

| 指令 | 說明 | 前提條件 |
|------|------|---------|
| `ingest SS-XXXX-N` | 將已驗證題目寫入 wiki | `verificationStatus = "verified"` |
| `ingest all` | 批次 ingest 所有 verified 但尚未進 wiki 的題目 | 同上 |
| `compile all` | 從零重建整個 wiki（慎用） | — |
| `rebuild [目錄]` | 重建特定 wiki 目錄（如 rebuild concepts） | — |
| `reindex` | 掃描 raw/solutions/ 修正 hasSolution 不一致 | — |
| `add concept [概念名]` | 新增概念到 concepts.json 並建立 wiki 頁面 | — |
| `add method [方法名]` | 新增方法論到 raw/solutions/methods/ | — |
| `lint wiki` | 完整 17 項健檢（[error] / [warn] 分級報告） | — |
| `status` | 驗證進度與統計（X/98 + 標籤統計） | — |
| `unverified` | 快速列出有解析但尚未 verified 的題目 | — |
| `frequency` | 各考點歷年出現次數 + 高頻標籤 Top 15 | — |
| `cross-ref [concept-id]` | 列出所有涉及某概念的題目 | — |
| `update diagnosis` | 根據已解題目更新診斷決策樹 | — |
| `update failure-modes` | 更新失敗模式層 | — |
| `update materials` | 更新材料行為層 | — |
| `更新儀表板資料` | 從 question_index.json 重新生成 dashboard-data.js | 任何題目或 viz 有更新後執行 |

## 6. 常見問題

**Q：解析 .md 放在哪裡？**
- 原始來源（唯讀）：`raw/solutions/SS-XXXX-N/SS-XXXX-N.md`
- 知識庫版本（ingest 後）：`wiki/problems/SS-XXXX-N.md`

**Q：如何更正已 verified 的答案？**
告訴 Cowork：「將 SS-XXXX-N 的 verificationStatus 改為 needs-review」→ 說明錯誤 → 重新驗算 → 改回 verified → 執行 `ingest SS-XXXX-N`。

**Q：新增的概念頁 / 方法頁需要 ingest 嗎？**
不需要。`wiki/concepts/`、`wiki/methods/`、`wiki/code-ref/` 等由 Cowork 直接維護，不走 ingest。

**Q：截圖要怎麼命名？**

| 類型碼 | 內容 | 範例 |
|--------|------|------|
| `fig` | 題目附圖 | `SS-2015-1-fig-1.png` |
| `chart` | 設計圖表 | `SS-2015-1-chart-1.png` |
| `eqn` | 公式截圖 | `SS-2015-1-eqn-1.png` |
| `hand` | 手寫補充 | `SS-2015-1-hand-1.png` |

**Q：誰負責什麼？**

所有操作均在 **Cowork** 執行；使用者只需負責截圖與人工驗算。

| 任務 | 執行者 |
|------|:------:|
| 解析考題（SOLVE） | Cowork |
| 新增知識點、add concept、add method | Cowork |
| VHA 變數層次分析 | Cowork |
| ingest / ingest all / compile all / rebuild | Cowork |
| lint / status / frequency / unverified | Cowork |
| 題目附圖截圖（fig/eqn/chart/hand）、人工驗算 | 使用者 |
